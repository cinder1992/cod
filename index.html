Cod

Python Extended IRC Services

Download as .zip Download as .tar.gz View on GitHub

Abstract

Cod is an extended IRC services package designed to solve a few accountability and convenience issues with existing IRC services packages.

Features

  • DNSBL lookups by nick or IP address This is useful in support for speed and accuracy of support.
  • Forwarding of non-services KILLs to the snoop channel This is useful in making operators more easily accountable for their actions.

Installation

  1. Extract cod to some directory
  2. Copy the example configuration and edit it to your needs, the defaults are pretty sane.
  3. Add the appropriate connect block to Charybdis, ponychat-ircd or any other ircd that uses TS6. Your mileage may vary.
  4. Change to the source directory src
  5. python cod.py
  6. Watch it link up and enjoy your new IRC services!

Usage

Commands in channels are prefixes by the prefix set in me::prefix.

List of Modules

Core modules

Name Description
core / s2scommands Defines handlers for TS6 commands. Do not unload this.
core / admin Defines commands for module loading and channel joinparts. Do not unload this.
protocol / ponychat-ircd Defines the login process for ponychat-ircd. Do not unload this.

Normal modules

Name Description
dnsbl Does searching in DNS blacklists on clients by nick or arbitrary IP addresses. Usage is RBL TARGET.
dnsblannounce Announces DNSBL hits to the services snoop channel and prints a full lookup on the hit to the services snoop channel.
killannounce Announces all non-services sourced kills to the services snoop channel with the reason
motd Handles a remote /MOTD with a customizable text blurb.
mpdclient Connects to an MPD server and allows users to search the database or print basic statistics.
ofc Orbital Friendship Cannon. This module is designed to stress test the IRC daemon you are connected to.
opname Generates military operation names that would have come from a 1980's B movie. Usage is OPNAME.
prettyprint Prints channel messages with the nickname source when services is not running with etc::production set to true.
rehash Listens for remote REHASH commands and respects them.
say Sends an arbitrary message to an arbitrary channel.
stats Shows stats on a remote STATS command.
test Example module for use of creating modules.
version Replies with its version number.
